## v1 を触り直して気づいた違和感（ステップ1）

- v1 のデモは見た目は良いが、**ボタンを押しても実際には何も起きない mock**。本当には使えない。
- もし本当に「全自動で片づける」が動いたら、**勝手に動かされて怖い**。信用できない。
- 分類は **名前・拡張子・日付だけ**。中身を見ないので、名前が違う重複や中身が違う同名ファイルを扱えない。
- 散らかりは1フォルダの問題ではない。実際は **Downloads / Desktop / Documents に分散して重複** している。
- → **「整理が進まない」本当の理由は、面倒さより「壊すのが怖い」だった。** 整理ツールに必要なのは賢さより**信頼**。

## v2 で実装したこと（v1 → v2 差分）

| 観点 | v1（mock） | v2（本物 = tidy） |
|---|---|---|
| 実体 | 見た目だけ・動かない | 公開スキル・テスト35項目合格 |
| 分類 | 名前・拡張子・日付 | **ファイルの中身（スニペット）も読む** |
| 安全性 | なし（消えたら終わり） | **削除しない隔離（_捨て）＋ 実行前プレビュー ＋ 全操作を記録して undo ＋ 中断耐性** |
| 範囲 | 1フォルダ・重複は印だけ | **複数の場所から集約＋同一内容（sha1）の重複を実際に排除** |
| 事故防止 | なし | **assess で .git / コードプロジェクト / システム配下を判定して止める** |

一言でいうと: **「全自動で片づける（が怖くて押せない）」→「消さない・見せる・戻せるから、安心して押せる」。**

## ローカルで動かす

```bash
# Web プロトタイプ v2
open lectures/lecture6/prototype/index.html

# 実装（本物）をインストールして使う

ln -s "$(pwd)/tidy" ~/.claude/skills/tidy
# あとは Claude Code に「このフォルダを整理して」「講義資料をまとめて」と頼む
```
