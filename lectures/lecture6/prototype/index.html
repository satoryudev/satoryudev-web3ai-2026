v5 まで、記録は各フォルダの _整理ログ/ に残っていた。でも場所ごとに分散していて、通しで振り返れなかった。

"どこを・いつ・何を動かしたか分からなくなる。1か所で振り返れないの？
記録は自動
整理のたびに1行、ホームへ集約
apply / undo / redo / review が成功するたび、その1行が ~/.tidy/history.jsonl に追記される。どのフォルダを整理しても履歴は1本に集まる。書けなくても整理自体は成功（ベストエフォート）。
{"ts":"...","action":"apply",
 "target":"~/Downloads","moves":8,"trashed":2}
history
横断タイムライン — 新しい順に
いつ・何を・どこで、を記号ラベル付きで一望。場所をまたいで「先週 tidy で何やったっけ」に答えられる。
tidy history
06-24 17:24 ✕ 物理削除 12件(4.2MB) @Documents
06-23 09:10 ▸ 整理 8件(隔離2) @Downloads
history --target
特定フォルダだけに絞る
「あのフォルダ最後にいつ整理した？」に一発。--verbose で manifest / ログのパスも辿れる。
tidy history --target ~/Downloads
記録数: 2 件（Downloads だけ）
実ファイルは動かさない — 索引だけホームへ
整理はローカル（rename）のまま、履歴だけを1か所に
「散らかりの整理」は従来どおり同一ボリューム内の rename で行い、ホーム ~/.tidy に置くのは来歴の索引だけ。外付けドライブでも速度・安全性は変わらない。TIDY_HOME で保存先を差し替え可能（この履歴の上に、将来 Obsidian への書き出しも載せられる）。

v5 のキモ: review — _捨て の lifecycle

「削除しない」を貫いた先の、
運用の不安に答える。

v2 から「削除しない代わりに _捨て/ に隔離する」を貫いてきた。v4 を使ってもらった次に出てきたのは、その帰結への不安だった。

"_捨て にどんどん溜まる気がする。後で見返したり片付けたりするフローが欲しい
review
一覧 — 来歴つきで全部見える
manifest を横断して、各ファイルの隔離日時・元の場所・理由・経過日数を辞書化して表示。manifest 未追跡（手で置かれた）も別表記で区別。
tidy review ~/Documents
.DS_Store (0 B, 3d 前)
  理由: システム生成ファイル
  元の場所: ~/Downloads/.DS_Store
review --restore
復元 — pattern で個別に戻す
glob 一致の隔離ファイルを manifest の from に従って元の場所へ。apply と同じ衝突回避（dedupe rename）。--yes 必須、無しなら dry-run。
tidy review ~/Documents --restore "*.pdf" --yes
復元完了: 3 件を元の場所へ戻しました。
review --purge
物理削除 — 古いものだけ、明示の同意で
tidy 唯一の delete 経路。--older-than DAYS で日数より古いものだけ対象。--yes 必須、無しなら件数とサイズだけ提示して停止。purge ログを別ファイルに残す。
tidy review ~/Documents --purge --older-than 30 --yes
物理削除: 12 件 (4.2 MB)
記録: ~/Documents/_整理ログ/purge-...jsonl
継続利用へのループが閉じた
整理 → 確認 → 片付け、すべて自分で完結できる
apply（v2〜）→ verify（v3〜）→ undo / redo（v3〜）→ review / restore / purge（v5）。「削除しない」「戻せる」「貯めたものを自分で扱える」のセット。すべて manifest と journal に記録されるので、後から「誰が・いつ・何を・なぜ動かしたか」を辿れる。

v3 のキモ: 依存解析クラスタ

import を読むから、
関係するファイルを切り離さない。

scan が各コードファイルを開いて import / require / src= / @import を抽出 → 連結したファイル群を1つのクラスタとして scan.json に記録。suggest はクラスタを尊重して同じ subfolder に配置する。手で plan を変えてクラスタを分断すると preview が止める。

🎨style.css—
🖼logo.png—

01 v6 NEW
history — 横断タイムライン
tidy history で全ての場所の操作を新しい順に一望。apply / undo / redo / review が ~/.tidy/history.jsonl に集約され、「いつ・何を・どこで」を記号ラベル付きで表示。

      
02 v6 NEW
--target フィルタ / 索引だけ集約
--target <dir> で特定フォルダの操作だけに絞る。実ファイルは動かさず来歴の索引だけをホームへ。TIDY_HOME で保存先も差し替え可能。

      
03 v5〜
review — _捨て の見回り
隔離ファイルを一覧（隔離日時・元の場所・理由）、--restore で復元、--purge --older-than 30 --yes で古いものだけ物理削除（唯一の delete 経路）。

      
04 v4〜
Downloads → ホーム配下への振り分け
scan ~/Downloads / suggest --target ~/Documents が素直に動く。cross_target 判定を「単一 root でも target が異なる場合」に拡張。

      
05 v3〜
依存解析クラスタ
Python の import、JS の require / import、HTML の src= / href=、CSS の @import を読む。連結したファイル群は同じ subfolder へまとめて配置。

      
06
削除しない隔離 + suggest / verify / redo（v2〜）
不要・重複ファイルは _捨て/ へ、assess で危険な対象は止める。plan は scan から自動生成、apply 後の整合性チェック、undo / redo / Ctrl-C 安全終了で「やり直し」のすべての方向に対応。

VPC v1 · Pain → v6 の答え

      

        v1 は 見せる、v2 は 怖くないから押せる、v3 は 関係性も壊さない、v4 は 普通の使い方が普通に動く、v5 は 貯めたものを自分で完結できるを作った。
        v6 は 「積み重ねを一望できる」を実装した — 1回の整理は完璧でも、複数の場所を整理すると「どこを・いつ・何を」が分散して見えなくなっていた。
        history で全ての操作を1本のタイムラインに集約し、場所をまたいで振り返れる。実ファイルは動かさず、来歴の索引だけをホームに。
        使い続けてもらうほど、機能そのものより「安心して任せ続けられる土台（振り返れる・辿れる）」への要望が出てくる。
