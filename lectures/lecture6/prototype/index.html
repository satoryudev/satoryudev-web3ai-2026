片づけたら、import が壊れた。

v2 で 「怖くないから押せる」を作った。コードフォルダに当てたら次の問題が出た: main.py と helper.py が別フォルダに振り分けられて import が壊れた。 消さなくても、関係性が切れたら壊れる。Tidy v3 は 依存関係を読んで、 連結しているファイル群を 1つのクラスタとして一緒に運ぶ。

「消さない」だけでは、
足りなかった。

v2 で「怖くないから押せる」が完成して、実際にコードフォルダに当ててみた。ファイルは生きてる。でも 関係性が切れて、プログラムが動かなくなった。「壊さない」の盲点だった。

v2 まででは足りなかった
  • main.py と helper.py が別フォルダに分かれる
  • HTML と参照している CSS / JS / 画像がバラける
  • 消さなくても import が壊れて動かなくなる
  • plan を毎回ゼロから書く負担
  • 整合性確認は手作業（apply 後ちゃんと動いた？）
v3 で入れたこと
  • 依存解析: import / require / src= / @import を読む
  • 連結クラスタは同じ subfolder に一緒に運ぶ
  • クラスタを分断する plan は preview で警告
  • suggest: scan.json から baseline plan を自動生成
  • verify: apply 直後の整合性チェック / redo: undo の取り消し / Ctrl-C 安全終了

v3 のキモ: 依存解析クラスタ

import を読むから、
関係するファイルを切り離さない。

scan が各コードファイルを開いて import / require / src= / @import を抽出 → 連結したファイル群を1つのクラスタとして scan.json に記録。suggest はクラスタを尊重して同じ subfolder に配置する。手で plan を変えてクラスタを分断すると preview が止める。

main.py ─ from helper import x → helper.py ／ page.html ─ <script src="./app.js"> → app.js ─ import './bar' → bar.js
v2 までは… 関係を見ずに型で分類 → import 破壊
コード/
├ main.py     ← from helper import x が壊れる
├ helper.py   ← 別の場所へ振り分けられた
└ page.html   ← src="./app.js" が解決できない
画像/
└ logo.png    ← page.html から参照されてたのに分離
v3 は… クラスタごとに同じ subfolder へ
コード/
├ main/           ← クラスタ「main」
│  ├ main.py
│  ├ helper.py
│  └ utils.py
└ page/           ← クラスタ「page」（HTML+CSS+JS+画像が同居）
   ├ page.html
   ├ app.js
   ├ bar.js
   ├ style.css
   └ logo.png
対応する依存元: Python（dot relative + sibling）／ JS/TS/JSX/TSX（import / require / 動的 import / 拡張子補完）／ HTML（src/href）／ CSS（@import / url）／ Markdown（画像参照）／ Shell（source）。外部パッケージ（react, os）は解決不能で自然に除外。
安全装置: assess（診断）

「触らない方がいい場所」は、止める。

整理を始める前に対象フォルダを診断。コードプロジェクトやシステム配下を誤って壊す事故を防ぐ。フォルダを選んでみてください。

v3 で動くもの

関係を読む。
関係するものは、一緒に運ぶ。

01 v3 NEW

依存解析クラスタ

Python の import、JS の require / import、HTML の src= / href=、CSS の @import を読む。連結したファイル群は同じ subfolder へまとめて配置。

02 v3 NEW

クラスタ分断警告

手で plan を編集して main.py と helper.py が別フォルダに分かれたら preview が止める。import が壊れる前に気付ける。

03 v3 NEW

suggest（自動 baseline）

scan.json から plan を自動生成。人 / Claude はゼロから書かず、差分修正だけで済む。

04 v3 NEW

verify / redo / Ctrl-C 安全終了

apply 後の整合性チェック、undo の取り消し、中断時の journal フラッシュ。「やり直し」のすべての方向に答える。

05

削除しない隔離（v2〜）

不要・重複ファイルはゴミ箱ではなく _捨て/ へ。assess で危険な対象は止める。

06

集約＋重複排除（v2〜）

複数の場所に散らばった資料を1か所に集め、同一内容（sha1一致）の重複を1つに絞る。

VPC v1 · Pain → v3 の答え
v1 は 見せる だけだった。v2 は 怖くないから押せる（消さない・戻せる・見せる）を作った。 v3 は 関係性も壊さない を入れた — main.py と helper.py、HTML と CSS/JS/画像、それぞれの「束」を ばらさず一緒に運ぶ。整理した結果、動かなくなるのは「壊さない」の盲点だった。
